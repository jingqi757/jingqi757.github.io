pwn45

32位recv

32位栈溢出，32位最后得到的b’xf7’，64位则是b’x7f’，recvuntil期望得到字节，所以加上b’’以免报错

pwn46

64位puts堆栈平衡

在64位puts对堆栈平衡要求不严，加了ret反而会崩溃，这里看别的师傅写的，具体原因我还不知道

[-6:].ljust(8,b’\x00’)是因为64位用16位十六进制表示，只有12位不为0，而recvuntil收到\x00会停止，并且u64只能接受64位，所以要补’\x00’

pwn47

%p输出不用u32，u64接收

在其他题都是用u32，这里用了u32却会报错，只能用eval或者是int，eval，eval期望接受的是字符串，int可以接受的比较多，但是recvuntil接受和输出的却都是字节，drop=True可以不接受最后的\n,decode可以把字节变成正确的ascii，这里都是字符

在写的时候我很疑惑，既然ida显示的输出%p，为什么还要recvuntil(b”\n”)呢，因为计算机传输是字节串，地址在内存里存的是字节：\x40\x23\xd1\xf7（小端序），在%p的情况下输出的是10个ascii，所以不能用u32，u32只能用于四个字节

pwn49

构造多个payload需要pop参数+静态编译，不能泄露libc

32位是不用pop到寄存器里面的，这道题一开始是没有用三个pop，但是之前题都没有pop掉参数是因为后面没有函数，已经是最后一个，但是这个后面还会ret一个函数，所以要pop掉参数，不然结构混乱，既是参数也是第二个函数的参数，还是第二个函数的返回地址

canary

pwn54

字符串拼接漏洞

字符串拼接溢出一开始想用，后来发现不行，用到puts：遇到第一个 \0（字符串结束符）才停止；此外，输出完成后会自动追加一个 \n（换行符），但是这一题有strcat会去掉\0，所以puts就可以一直输出而且v5上面就是存储密码的s，就可以输出password

还有一个与本题无关的，就是题上有这个 v8 = strchr(v5, 10); if ( v8 ) *v8 = 0;怎么知道v8是整数0还是ascii 0呢，可以看v8占的字节数，这里是一个，所以就是ascii 0
